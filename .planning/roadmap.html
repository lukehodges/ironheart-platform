The Rule
Before writing any new feature: does it belong in an engine? → add a config option, not a code path. Does it introduce a client-specific check? → make it a table row. If a future client asks for something genuinely new, the question is: "which engine needs a new rule type?" — not "which module do I add client X logic to?"
