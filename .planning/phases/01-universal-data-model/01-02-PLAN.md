`resources.types.ts`:
```typescript
export type ResourceType = 'PERSON' | 'VEHICLE' | 'ROOM' | 'EQUIPMENT' | 'VIRTUAL'
export type AssignmentRole = 'LEAD' | 'SUPPORT' | 'DRIVER' | 'OBSERVER'

export interface Resource {
  id: string
  tenantId: string
  type: ResourceType
  name: string
  slug: string
  capacity: number
  homeAddressId: string | null
  travelEnabled: boolean
  skillTags: string[] | null
  userId: string | null
  isActive: boolean
  metadata: Record<string, unknown> | null
  createdAt: Date
  updatedAt: Date
}

export interface Address {
  id: string
  tenantId: string
  line1: string
  line2: string | null
  city: string
  county: string | null
  postcode: string
  country: string
  lat: string | null
  lng: string | null
  geocodedAt: Date | null
  label: string | null
  createdAt: Date
}

export interface CreateResourceInput {
  type: ResourceType
  name: string
  slug?: string
  capacity?: number
  travelEnabled?: boolean
  skillTags?: string[]
  userId?: string | null
  metadata?: Record<string, unknown>
}

export interface UpdateResourceInput extends Partial<CreateResourceInput> {
  isActive?: boolean
  homeAddressId?: string | null
}

export interface ListAvailableInput {
  date: Date
  startTime: string
  endTime: string
  skillTags?: string[]
  type?: ResourceType
}
```

`resources.schemas.ts` (Zod schemas for tRPC input validation):
- `createResourceSchema`: z.object({ type: z.enum([...ResourceTypes]), name: z.string().min(1).max(100), ... })
- `updateResourceSchema`: createResourceSchema.partial().extend({ isActive: z.boolean().optional() })
- `listResourcesSchema`: z.object({ type: z.enum([...]).optional(), isActive: z.boolean().optional(), limit: z.number().int().max(100).default(50), cursor: z.string().optional() })
- `listAvailableSchema`: z.object({ date: z.string(), startTime: z.string(), endTime: z.string(), skillTags: z.array(z.string()).optional(), type: z.enum([...]).optional() })

`resources.repository.ts`:
- findById(tenantId, resourceId): select from resources where id AND tenantId, limit 1
- findByUserId(tenantId, userId): select where userId AND tenantId
- list(tenantId, filters): select with optional type/isActive filters, pagination (limit+1 pattern)
- create(tenantId, input): insert returning; auto-generate slug from name if not provided (lowercase, replace non-alphanumeric with `-`)
- update(tenantId, resourceId, input): update where id AND tenantId, returning
- delete(tenantId, resourceId): soft delete (set isActive = false) OR hard delete — use hard delete since resources replaces staff profiles
- listAvailable(tenantId, input): find resources not assigned to overlapping jobs on that date/time window; query jobAssignments JOIN jobs to find busy resources, then exclude them

`resources.service.ts`:
- Thin layer calling repository; throws NotFoundError when resource not found
- logger.child({ module: 'resources.service' })
- Procedures: create, update, getById, list, delete, listAvailable

`resources.router.ts`:
- All procedures are `tenantProcedure` (authenticated + tenant context)
- `create`: permissionProcedure('resources:create') — input: createResourceSchema
- `update`: permissionProcedure('resources:update') — input: z.object({ id: z.uuid(), data: updateResourceSchema })
- `getById`: tenantProcedure — input: z.object({ id: z.uuid() })
- `list`: tenantProcedure — input: listResourcesSchema
- `delete`: permissionProcedure('resources:delete') — input: z.object({ id: z.uuid() })
- `listAvailable`: tenantProcedure — input: listAvailableSchema

`index.ts`:
```typescript
export * from './resources.types'
export * from './resources.schemas'
export { resourceRepository } from './resources.repository'
export { resourceService } from './resources.service'
export { resourcesRouter } from './resources.router'
```
