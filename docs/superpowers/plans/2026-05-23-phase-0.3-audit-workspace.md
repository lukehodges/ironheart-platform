## Goal

Build the 3-layer audit workspace Luke uses to capture call notes, score lenses, log findings, draft recommendations. Backend (`audit-workspace` module) already exists per file structure — UI is the missing piece.

Lives at `/platform/clients/[id]/audit`.

---

## Three layers

### Layer 1 — Capture

- Team sidebar: contacts from org chart w/ form status + interview booking status
- Per-person notes pane (one tab per contact)
- Form responses shown as read-only highlights above notes
- Auto-save (debounced 500ms)

### Layer 2 — Processing

- 5 lens tabs: REVENUE / OPERATIONS / FINANCE / TECHNOLOGY / TEAM
- Per lens:
  - RAG score selector (3 big buttons)
  - Current-state editor (textarea, auto-save)
  - Findings table: text + impact + evidence + priority + £ annual waste (inline edit, drag-reorder)
  - Recommendations list: action + effort + cost + priority

### Layer 3 — Report Ready

- Validation indicator: all 5 lenses scored + ≥1 finding each
- "Generate Report" CTA → kicks off 0.4 flow
- Lock state once report drafted (prevent edits during review)

## Open questions (resolve before starting)

1. Auto-save granularity — per-field or per-section? Per-field is heavier on tRPC, per-section may lose work.
2. RAG justification — required when score is RED? Or always optional?
3. Findings drag-reorder — `@dnd-kit/sortable` (already in stack) or simpler up/down buttons?
4. Multi-user editing — if Luke + Sam both auditing same engagement, optimistic concurrency or last-writer-wins?

## Success criteria

- Luke conducts a real audit fully in the platform
- All 5 lenses scored, ≥3 findings + ≥2 recommendations each
- "Generate Report" enabled, no errors
- No external tools (no Notion, no spreadsheets) used during the audit
