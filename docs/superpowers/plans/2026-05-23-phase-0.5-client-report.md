# Phase 0.5 — Client Report View + Walkthrough Booking

> **Parent roadmap**: [`ROADMAP.md`](./ROADMAP.md)
> **Status**: 📋 PLANNED
> **Depends on**: Phase 0.4 (published report exists)
> **Estimate placeholder**: ~2 days

---

## Goal

Client logs into portal, sees their audit report inside the platform (not as PDF attachment in email). Reads exec summary, RAG scores, findings + £ waste, full report. Books walkthrough call from the report page. Optional implementation proposal CTA.

This closes the loop. After 0.5, the entire audit lifecycle lives inside the platform.

## UI sections (top to bottom)

1. **Header** — engagement title, "Audit Report", published date, download PDF button
2. **Exec summary** — top 3 findings + total estimated annual waste (serif number, big)
3. **5-lens RAG strip** — visual summary (5 colored cards w/ score)
4. **Per-lens expandable sections** — current state, findings, recommendations
5. **Walkthrough CTA** — "Book a walkthrough call with Luke" → opens Cal.com 30-min slot embed
6. **Implementation proposal CTA** — "Ready to fix what we found? Get an implementation proposal" → emits `engagement/proposal-requested` event (Luke gets notified, builds proposal in Phase 1.x)

## Success criteria

- Client receives email notification when Luke publishes
- Clicks link → lands logged-in on `/[tenantSlug]/dashboard/report`
- Reads full report, downloads PDF
- Books walkthrough call directly from page
- Optionally requests implementation proposal
- Luke sees activity in his platform: "Northwind viewed report, booked walkthrough Tue 14:00"
