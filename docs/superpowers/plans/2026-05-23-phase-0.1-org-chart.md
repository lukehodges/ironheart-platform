## Goal

Build the collaborative org chart that drives audit interview routing. Two surfaces — consultant `/platform/clients/[id]/onboarding` + client `/[tenantSlug]/dashboard/onboarding` — sharing one source of truth. Both sides can read + write, activity log surfaces changes.

This is the headline feature of 0.1. If it works, the platform↔subtenant integration is proven.

## Schema

### New table: `engagementOrgChart`

```typescript
export const engagementOrgChart = pgTable("engagement_org_chart", {
  id: uuid("id").defaultRandom().primaryKey(),
  tenantId: uuid("tenant_id").notNull(),         // for tenant isolation
  engagementId: uuid("engagement_id").notNull().references(() => engagements.id, { onDelete: "cascade" }),
  parentId: uuid("parent_id"),                   // self-ref nullable. null = root.

  label: text("label").notNull(),                // "Warehouse" or "Sarah Chen — Ops Lead"
  type: text("type", { enum: ["DEPARTMENT", "ROLE", "PERSON"] }).notNull(),

  headcount: integer("headcount"),               // null unless DEPARTMENT
  contactUserId: uuid("contact_user_id"),        // null unless PERSON + assigned
  contactEmail: text("contact_email"),           // null unless PERSON
  contactName: text("contact_name"),
  contactRole: text("contact_role"),

  interviewMode: text("interview_mode", { enum: ["ALL", "SAMPLE", "OWNER_ONLY", "SKIP"] })
    .notNull().default("OWNER_ONLY"),
  sampleSize: integer("sample_size"),            // null unless mode=SAMPLE
  templateSlugOverride: text("template_slug_override"),  // null = auto from role

  sortOrder: integer("sort_order").notNull().default(0),
  version: integer("version").notNull().default(1),       // optimistic concurrency
  lastEditedBy: text("last_edited_by", { enum: ["CONSULTANT", "CLIENT"] }).notNull(),
  lastEditedAt: timestamp("last_edited_at", { withTimezone: true }).notNull().defaultNow(),

  createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp("updated_at", { withTimezone: true }).notNull().defaultNow(),
}, (t) => ({
  byEngagement: index("idx_org_chart_engagement").on(t.engagementId),
  byParent: index("idx_org_chart_parent").on(t.parentId),
  byTenant: index("idx_org_chart_tenant").on(t.tenantId),
}))
```

### New table: `engagementOrgChartActivity`

```typescript
export const engagementOrgChartActivity = pgTable("engagement_org_chart_activity", {
  id: uuid("id").defaultRandom().primaryKey(),
  engagementId: uuid("engagement_id").notNull(),
  nodeId: uuid("node_id"),                        // nullable — some events span chart
  actorType: text("actor_type", { enum: ["CONSULTANT", "CLIENT", "SYSTEM"] }).notNull(),
  actorId: uuid("actor_id"),                      // userId
  actorName: text("actor_name").notNull(),        // denormalised for fast read
  action: text("action").notNull(),               // "node.created", "node.updated", "node.deleted", "person.assigned", "mode.changed", "chart.seeded", "plan.approved"
  fromValue: jsonb("from_value"),                 // optional diff
  toValue: jsonb("to_value"),
  message: text("message").notNull(),             // human-readable, e.g. "Sarah Chen named for Ops sample slot"
  createdAt: timestamp("created_at", { withTimezone: true }).notNull().defaultNow(),
}, (t) => ({
  byEngagement: index("idx_org_chart_activity_engagement").on(t.engagementId, t.createdAt.desc()),
}))
```

// src/modules/onboarding/onboarding.types.ts (NEW module — see note below)

export type OrgChartNodeType = "DEPARTMENT" | "ROLE" | "PERSON";
export type InterviewMode = "ALL" | "SAMPLE" | "OWNER_ONLY" | "SKIP";
export type AuditTier = "MICRO" | "SMALL" | "MID" | "LARGE";
export type ChartActorType = "CONSULTANT" | "CLIENT" | "SYSTEM";

export interface OrgChartNodeRecord {
  id: string;
  tenantId: string;
  engagementId: string;
  parentId: string | null;
  label: string;
  type: OrgChartNodeType;
  headcount: number | null;
  contactUserId: string | null;
  contactEmail: string | null;
  contactName: string | null;
  contactRole: string | null;
  interviewMode: InterviewMode;
  sampleSize: number | null;
  templateSlugOverride: string | null;
  sortOrder: number;
  version: number;
  lastEditedBy: "CONSULTANT" | "CLIENT";
  lastEditedAt: Date;
  createdAt: Date;
  updatedAt: Date;
}

export interface OrgChartTree extends OrgChartNodeRecord {
  children: OrgChartTree[];
}

export interface ChartActivityRecord {
  id: string;
  engagementId: string;
  nodeId: string | null;
  actorType: ChartActorType;
  actorId: string | null;
  actorName: string;
  action: string;
  message: string;
  fromValue: unknown;
  toValue: unknown;
  createdAt: Date;
}

export interface PlannedFormSend {
  nodeId: string;
  contactEmail: string;
  contactName: string;
  templateSlug: string;
  reason: string;  // e.g. "OWNER role, MICRO tier"
}

export interface OnboardingPlan {
  engagementId: string;
  tier: AuditTier;
  totalSends: number;
  sends: PlannedFormSend[];
  unfilledSampleSlots: { nodeId: string; deptLabel: string; needed: number }[];
}

export interface OnboardingStatus {
  totalRecipients: number;
  sent: number;
  completed: number;
  pending: number;
  byContact: { name: string; email: string; status: "PENDING" | "SENT" | "COMPLETED" }[];
}

### Task 3: Service — tier + seeding

**Files**: `src/modules/onboarding/onboarding.service.ts`

- [ ] `resolveTier(teamSize: number | null): AuditTier`:
  - null or <5 → MICRO
  - 5–15 → SMALL
  - 15–50 → MID
  - >50 → LARGE
- [ ] `seedChartFromTier(ctx, engagementId)`:
  - Read engagement's `qualificationData.teamSize`
  - If chart already has nodes → no-op (idempotent)
  - Resolve tier
  - Create root DEPARTMENT node (label = customer company name, headcount = teamSize)
  - Per tier, create child nodes:
    - MICRO: 1 PERSON (Owner) mode=ALL + 1 catchall ROLE "Other team" mode=OWNER_ONLY
    - SMALL: Owner ALL + 3 functional ROLEs (Operations, Finance, Sales) each ALL + 1 "Other staff" SAMPLE(2)
    - MID: Owner ALL + 4 functional dept DEPARTMENTs ALL + 1 "Other staff" SAMPLE(3)
    - LARGE: Owner ALL + functional heads ALL + dept buckets SAMPLE(1) each
  - Log activity: `chart.seeded` w/ tier
- [ ] Helper: `matchQuestionnaireTemplate(role)` — move from `consulting/onboarding.service.ts`, keep keyword logic
- [ ] `mapNodeToTemplate(node: OrgChartNodeRecord): string` — if `templateSlugOverride` set use it, else infer from `contactRole` or `label` via `matchQuestionnaireTemplate`

### Task 4: Service — plan + enqueue

**Files**: `src/modules/onboarding/onboarding.service.ts`

- [ ] `planOnboardingForms(ctx, engagementId): OnboardingPlan`:
  - Read chart tree
  - Walk tree depth-first
  - For each node:
    - mode=SKIP → ignore
    - mode=OWNER_ONLY → skip unless this is the root owner node
    - mode=ALL + PERSON → add 1 send w/ contactEmail
    - mode=ALL + ROLE/DEPARTMENT → ignore (need person under it)
    - mode=SAMPLE(N) + DEPARTMENT → if N child PERSON nodes exist, add their sends; if fewer than N → record unfilledSampleSlots
  - Return plan + unfilled slots warning
- [ ] `enqueueOnboardingForms(ctx, engagementId)`:

- [ ] `getOnboardingStatus(ctx, engagementId): OnboardingStatus`:
  - 0.1: return zeros until forms wired

## Risks

- **Tree depth blowups** — no soft cap on tree depth. Add a check at 5 levels max to prevent UI getting wild. Most orgs are ≤3.
- **Race conditions on collaborative edit** — version column + 409 Conflict response handled gracefully in UI ("Client just made a change, refresh"). Don't over-engineer w/ Y.js for 0.1.
- **Activity log noise** — every keystroke shouldn't log. Debounce label/text edits server-side (only log when value differs from previous + 5s settled). Mode/template/structural changes log immediately.
- **Tier defaults may suck in practice** — they're a guess. After 0.1 ships, capture which were edited per real client, iterate. Don't over-design v1.
- **`onboarding` module name collision** — there's already `src/modules/team/onboarding` (staff onboarding). Use `onboarding` at module root for client onboarding; rename the team one if needed.
